## 能学到什么

### 道（底层认知，一辈子受用）

**AI的真正瓶颈不是智能，而是数据**

- 可拉反复强调一个比喻：「AI就像一个很厉害的厨师，如果他的食材散落在不同的地方——在市场、在冰箱、在邻居家——他做不出一盘好菜」
- 他用了整整一段讲"拿数据库权限"——不是在讲技术，是在讲**CEO的第一要务已经变了**：从"找人做事"变成"把数据收起来让AI做事"
- 最难打通的不是技术，是微信生态里的供应商沟通数据——「供应商都用微信，信息就停在微信里，AI拿不到这个数据就没法做判断」
- **数据通了，一切皆有可能。缺的东西从头到尾都是数据。**

**AI不是缩短你的工作时间，而是扩大你的能力边界**

- 这是他用AI之后最反直觉的发现：「你以为用了AI可以10分钟做完原来2小时的事，但实际上你会发现你的能力边界大幅提升，以前做不到的事现在都可以做了。然后你就陷入痛并快乐着的强迫性重复」
- 他的原话：「我不跑代码就觉得是对生命的浪费」
- 这不是笑话，这是一个深刻的产业认知：**AI的价值不在于替你做已有的事，而在于让你做以前不可能做的事**。以前5万个SKU标签有错就算了——现在可以让AI全部重新打一遍

**AI是顶级聪明但缺经验的实习生，你要给他的是信息而非指令**

- 「AI做这件事需要人工核查，不是因为他笨，而是因为他没有你的信息。你只要给到他这个信息，他可以做得非常好」
- 「你只要告诉他一个错误，他可能能把里面100个类似错误全部都检查出来」
- 这是对人机协作关系的最精准定义：**你的角色不是监工，而是信息源**。你给AI越多的领域知识，它做得越好

**不用焦虑落下了多少——你终究会学会，这才重要**

- 他用了一个精彩的类比：「假设你从三年前穿越过来，没经历GPT 3.5时代，直接用GPT 5.4和Cursor开始学。我相信你很快就会跟三年前就用AI的人一样好」
- 「中国教育特别喜欢搞学前班，4岁就做加减乘除。重要吗？重要的是你终究会学会」
- **三年前学的方法现在已经完全不同了。先发优势在AI时代几乎不存在。**

### 法（方法论框架，可迁移）

**AI落地三阶段演进模型**


| 阶段  | 时代           | AI角色     | 典型应用              | 特征            |
| --- | ------------ | -------- | ----------------- | ------------- |
| 1   | Midjourney时代 | 小工具      | 生成图标、产品宣传图、批量元素   | 单点任务，需要人工PS修正 |
| 2   | GPT时代        | 文本生产力工具  | 本地化翻译、战略决策对话、营销方案 | 对话驱动，参与运营     |
| 3   | Cursor时代     | AI Agent | 数据库操作、全栈SEO、自动化流程 | 直接操控系统，不需要写代码 |


→ 关键转折：从"AI辅助人做事"到"AI直接做事，人负责审核和提供信息"

**AI Agent协作的置信度分层法**

1. AI完成全部产品标签/SEO/分类
2. AI对每条结果标注置信度（1-5）
3. 置信度4-5：直接通过
4. 置信度2-3：人工审核
5. 人工审核时告诉AI你的判断逻辑
6. AI用这个逻辑检查所有类似案例
7. 循环迭代，置信度整体上升

→ **核心洞察：你只要告诉AI一个错误，它能把100个类似错误全检出来**

**团队AI推广四步法**

1. **老板先学**：自己跟AI对话+实操，搞清楚AI能给公司带来什么变革
2. **找年轻人当先锋**：找一个聪明的、会打字的年轻人教他Cursor。2-3周后他就超过你了
3. **当面演示给中层看**：周会上现场演示AI解决他们的真实痛点。不讲理论，直接演示"你看以前你有这个痛点，现在直接搞定了"
4. **现场安装、一对一带**：最大的阻力是安装和搞定账号。现场帮他们装好，消除畏难情绪

→ 关键原则：「如果他打电话问我怎么做，我说你有没有问过GPT？你问过了还搞不懂，我们可以聊。你如果没有问过，你先去问AI」

**SOP进化路径**


| 阶段  | 方式          | 问题               |
| --- | ----------- | ---------------- |
| 1.0 | 写文档         | 没人看，容易过时         |
| 2.0 | 录视频         | 效果不好，还是靠人带人      |
| 3.0 | MD文件（Rules） | AI直接执行SOP，人类不需要看 |


→ **SOP不再是给人看的文档，而是给AI执行的指令集**

### 术（具体打法，可直接用）

**案例一：测评视频一键转图文**

- 输入中文视频地址 → Cursor插件调用Vision API逐帧读取 → AI理解视频+文案 → 生成中文图文稿 → 翻译英文 → API一键上传网站后台
- 10分钟完成，以前需要人工看视频+手动翻译+手动上传
- 结果：中文视频素材不够用了（生产效率倒逼上游供给）

**案例二：客服响应时间监控**

- 团队成员自己用Cursor做的Chrome插件
- 调后台数据 → 统计每个客服管理员的回复时间、对话轮数 → 公开排名
- 效果：「只要公布了，大家的平均回复时间就会立刻缩短，因为谁也不想垫底」

**案例三：物流渠道智能审核**

- 痛点：不同国家×不同渠道×不同限制×不同报价，每家供应商表格格式不统一，经常过期
- 解法：AI清洗多格式报价单 → 标准化写入Google Sheets → API导出做成Chrome插件 → 全员安装，输入国家直接查最优渠道
- 物流同事只需维护Google Sheets，插件自动同步

**案例四：AI Agent自动重打全站产品标签**

- 获取数据库权限 → 读取产品标题+描述+图片
- AI逐条上网核实（电影角色出处、品牌归属等）
- 标注置信度 → 人工处理低置信度数据 → AI学习修正逻辑 → 批量写入数据库
- 关键技术：AI自动编排低成本Agent并行处理读图任务（用便宜模型读图，省钱且快）

**案例五：全栈SEO优化**

- AI读取产品图片 → 生成Alt标签（以前从不写，因为图片太多���
- 分析每个产品的别名、买家常见拼写错误 → 写入隐藏关键词
- 优化产品标题和描述
- 一周内完成：3万SKU + 6万条SEO + 18万张图片标签 + 21万产品标签

**Cursor实战安全法则**

1. 小批量验证，先单品闭环再批量
2. 拿到数据库权限后第一件事：做备份
3. 生产环境只用最强模型（Opus 4.6），智能选项开到Max
4. 全员盯着AI做事，不要无人值守

## 跨内容隐藏连接

### 与王大仙的连接：Rules文件就是SOP的AI时代形态

可拉说「任何一个团队都会有SOP，以前写文档没人看，录视频也没人看。有了AI之后直接让AI执行SOP」。王大仙说「把操作流程写成Skill文件，AI直接执行」。两个人不约而同走到了同一个终点：**SOP的进化方向不是"写得更好让人更愿意看"，而是"写成机器能执行的格式让人不用看"**。

### 与易亚婷的连接：AI让你做以前不可能做的事

易亚婷说「AI让80分内容价值趋近于0」，可拉说「AI让我能力边界大幅提升，以前做不到的事现在都能做」。两个人看到的是同一个硬币的两面：**AI同时在压缩低端价值和拉伸高端可能性**。80分的内容变得不值钱了，但以前不可能做的5万SKU全站重打标签变得可能了。关键是你站在哪一端。

### 与刘智行的连接：用量级碾压的两种形态

刘智行用量级碾压内容赛道（500号×200篇/天），可拉用量级碾压运营效率（3万SKU×6万SEO/周）。底层逻辑一模一样：**AI的核心杠杆是把边际成本压到接近零**。区别在于，刘智行把这个杠杆用在了流量获取上，可拉用在了运营效率上。前者赚的是流量的钱，后者赚的是效率的钱。

### 全场最有价值的一句话

> 「AI就像一个很厉害的厨师，再厉害，如果食材散落在市场、冰箱、邻居家——做不出好菜。真正的差距不是手艺更灵活，而是把食材集中、清洗和分装。」

这句话把所有嘉宾的分享串了起来：易亚婷的"脑子副本"是在集中食材，王大仙的"飞书Agent+MCP"是在搭建食材仓库，刘智行的"8000字提示词"是在标准化食材处理流程。**AI时代最值钱的不是AI本身，而是你喂给AI的数据资产。**
